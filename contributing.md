# Contribution Guidelines

Thank you for your interest in contributing to Awesome Bahrain! This is a community effort maintained by [Bahrain.js](https://bahrain.js.org).

## Adding an Entry

Please ensure your pull request adheres to the following guidelines:

- Search previous entries before making a new one, as yours may be a duplicate.
- Make an individual pull request for each suggestion.
- Use the following format: `- [Name](link) - Description.`
- Additions should be added to the bottom of the relevant category.
- New categories or improvements to the existing categorization are welcome, but discuss them in an issue first.
- Link to the GitHub repo when one exists. Otherwise, link to the official website.
- Keep descriptions concise, clear, and objective.
- Start the description with a capital letter and end with a period.
- Do not include trailing whitespace.
- Check your spelling and grammar.
- The pull request should have a useful title like `Add Project Name`.

## Quality Standards

We aim to curate the **best** resources, not everything. Entries should be:

- **Active** — Not unmaintained, archived, or deprecated.
- **Relevant** — Connected to Bahrain's tech ecosystem.
- **Useful** — Provides real value to developers, entrepreneurs, or the tech community in Bahrain.

## Removing an Entry

If you notice a link is broken, a project is archived, or an entry no longer meets the quality standards, please open a pull request or issue to have it removed.

## Code of Conduct

By participating in this project, you agree to be respectful and constructive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/) code of conduct.
